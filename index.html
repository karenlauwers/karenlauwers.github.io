GitHub Hoe het werkt

Algemeen

/localWorkingDirectory = your code, your playground

Staging Area = temporary place holding changes before committing

Local Repo = local area with committed changes

Remote Repo = remote area on GitHub servers where you push your code to or pull new code from

git clone url = zo kopieer een lokale versie van een gedeelde remote repo

git checkout = als je met de code aan de slag gaat, zodat je in je local environment aan de slag gaat

git add = stage snapshot, je voegt de wijzigingen toe aan de staging area , als je -A toevoegt, dan zeg je: voeg alle untracked changes toe. Dit is de voorbereiding op de commit

git commit = committen, zo sla je op in je local repo. Als je er -m achter zet, dan moet er een message volgen: git commit -m

git pull = to integrate gegevens op de remote repo met de gegevens op jouw local repo - dat heb je nodig als je gaat samenwerken. Iemand heeft code aangepast en jij moet daar op verder werken, dan moet je die binnenhalen.

Je kan dan conflicten krijgen: als jij iets zus hebt gezet en de ander zo, dan moet je kiezen om die conflicten op te lossen.
